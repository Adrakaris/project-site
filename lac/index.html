Large Arithmetic Collider

Overview

The Large Arithmetic Collider (LAC) is a maths game with simple rules.

You are given a grid of numbers, which may be positive or negative, which may be off or on. Attached to each of the columns and rows is a target value, which the row or column must sum to. Furthermore, rows and columns can be rotated, kind of like a rubiks cube, with the number rotated off the board looping back to the other end.

The goal is to rotate the columns and rows, and then turn on or off cells, such that all the row and column sum targets are satisfied.

A simple version of the game is where the grid consists of only a single row, such as the one shown on the cover image:

-3 9 0 0 14
20 14 -3 9 3 1

To solve this, we would have to rotate the single row into the position shown below, and turn on the cells for 14, -3, and 9.

-3 9 0 0 14
20 -3 9 3 1 14

Of course, it is not as easy as just one row, you could get grids - and very large ones at that. Thus, this is certainly not trivial to solve by hand, and not particularly trivial to brute force by computer either.

Solver

Now that we know the task, the goal of the assignment was to make a solver in Haskell, which could, in reasonable time, either solve the problem, or say it cannot be solved. Haskell is a (lazy, statically typed,) pure, functional langauge, which makes programming in it much different from any imperative language, and whilst some things were exceedingly difficult (No print statements in the middle of execution) Haskell allows for some very elegant functions with high level abstractions. On top of that, it was often the case that once all type errors were resolved and the program compiled, it worked, first time.

Since this is a coursework, source code is not available.
